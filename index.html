What's the difference between "p" and "h"?

In short, the only distinct difference between a small "h" tag and a "p" tag is that the content of an "h" tag is bolded. The content of a "p" tag is not. That said, best practice is to use "h" tags to show the hierarchy of a web page.

Images

images are inserted with the "img" tag, this tag is a self-closing tag (does not need an end tag, but is still written with a forward slash at the end of the tag).
"Img" tags are written out with an attribute called "src" which is set to the source of the image.
The "alt" attribute can also be added to an image to display text when the curser is held over an image which has failed to load (I'm still not fully sure how this one works).

Videos

Videos are inserted with the "video" tag. This tag also uses the "src" attribute in the same way as images. It also has "width" and "height" attributes which set the size of the video displayed.
There is another attribute, "controls", which tells the browser to include basic controls on the video such as pause, play, and skip.

It is common practice to add the phrase "Video not supported" in between the opening and closing "video" tag, which will be displayed if the browser is unable to load the video.

HTML Document Standards

In order to tell the browser what kind of document it is running, a document type declaration can be used.
For HTML5 (the current standard for HTML), a document type declaration is written out as "!DOCTYPE html". This must be written out at the beginning of all HTML documents.
In order to open an HTML file, the code should be saved with an .html extension.



Immediately following the document type declaration, an "html" tag should be used. This tag should also be used to end the document. This ensures that all code written will be interpreted as HTML code.

Following the "html" tag, a "head" tag should be used. This tag will contain the metadata for the web page (information about the page which isn't directly displayed).
To reiterate, the opening and closing "head tag" will appear in the code before the "body" tag.

Metadata

The "title" tag determines the title of the web page. This is the text that will be displayed at the top of the browser

Forms

Forms can be used to collect and send information. Forms are used in polls, search queries, comments, and many other things.
If a text box is used, the "form" tag is often involved in some way.



To make a form, use the "form" tag. Inside the form tag, use the "action" attribute to determine where information is sent.
The "method" attribute is assigned an HTTP verb that is included in the HTTP request(eg:post, etc.)

To create an input field in the form, use the "input" tag. The input tag uses a variety of attribute to work as desired.
This includes the "type" attribute which determines what kind of input field is used, the "name" attribute, which is paired with the input when submitted (a "name" attribute is required for information to be sent),
and the "value" attribute which prefills the input field with a predetermined value.

To help users identify what a form input is for, use the "label" tag.
A label tag works similarly to any other text tag, but it is linked to the "input" tag using the "id" attribute(on the input) and the "for" attribute (on the label).

there is also a "step" attribute which adds an arrow to increase/decrease the value of the input by the value of the attribute. This is particularly useful for "number" fields.

Types of "Type"

  • "text" (a simple text field)
  • "password" (a text field where all input is visually replaced with asterisks or dots)
  • "number" (text field which restricts input to numbers)
  • "range" (creates a slider. This type requires a "min" and "max" attribute (and often uses the "step" attribute) to be defined in the "input" tag.
  • "checkbox" (creates a checkbox. To create a selection of multiple checkboxes, use a seperate "input" tag for each input, each with the same "name" attribute, but with seperate "label" tags and "value" attributes.
  • "radio" (works the same as "checkbox", except they allow for only one option to be marked.
  • "submit" (creates a button to submit the form. Use a "value" attribute to define the text displayed on the submit button.


Dropdown Lists

To create a dropdown list in a form use the "select" tag. To add options to the list use the "option" tag.
These options are visually defined by the text in between the opening and closing tag. The information sent by the code is defined by the "value" attribute.

Another option is to use the "datalist" tag. This tag is used in conjunction with a "text" type input.
This creates a dropdown list that can be searched by typing in the text field. It also allows any text typed in the text field to be submitted as a response.



To create a larger text field, use the "textarea" tag. This, in conjuction with a "rows" and "cols" attribute will allow to make a text field of any desired size.
Default text can be placed in a "textarea" text field by writing the desired text in between the opening and closing tag.

Form Validation

In forms, there are some restrictions or necessary inputs which can be enforced using a series a different attributes/tags.
This is a process called validation.

Types of Validation

  • Required fields - When a field on the form must be filled out you can add the "required" attribute.
  • Min/Max - Fairly self-explanatory, a minimum and maximum value for a field can be defined using the "min" and "max" attributes respectively
  • # of characters - To define the number of characters that can be used in a field, use the "minlength" and "maxlength" attribute.
  • regualar expression(regex) - Some inputs have specific guidelines for their inputs. The "pattern" attribute can help enforce those guidelines.
    The pattern attribute is defined by placing a selection of characters in brackets (ex: [0-9],[a-zA-Z0-9]) which defines which characters can be used. Followed by a value in curly brackets (ex:{5},{14,16},+) which defines the acceptable combinations of characters
    For clarification, the "14,16" in the example above defines that any combination of allowed characters can be used between 14 and 16 times. The "+" defines that any combo can be used any number of times (however this does not override the limit defined by the "min" and "max" attributes.
    Some examples of a defined "pattern" attribute include: "[a-z]+", "[0-9]{10}","[a-zA-z]{10,12}"
